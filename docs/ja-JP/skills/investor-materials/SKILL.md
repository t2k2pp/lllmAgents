# 投資家向け資料

一貫性があり、信頼性が高く、説明しやすい投資家向け資料を作成します。

## 有効化するタイミング

- ピッチデックの作成または改訂
- 投資家メモまたはワンページャーの作成
- 財務モデル、マイルストーン計画、資金使途表の作成
- アクセラレーターまたはインキュベーターの応募質問への回答
- 複数の資金調達資料を一つの信頼できる情報源に統一

## 基本ルール

すべての投資家向け資料は互いに整合性を保つ必要があります。

作成前に、以下の項目について単一の信頼できる情報源を作成または確認してください：
- トラクション指標
- 価格設定と収益の前提条件
- 調達額と調達手段
- 資金使途
- チームの経歴と役職
- マイルストーンとタイムライン

矛盾する数値が見つかった場合は、ドラフト作成前に解決してください。

## コアワークフロー

1. 基準となる事実の棚卸し
2. 不足している前提条件の特定
3. 資料タイプの選択
4. 明示的なロジックに基づいた資料のドラフト作成
5. すべての数値を信頼できる情報源と照合

## 資料別ガイダンス

### ピッチデック
推奨フロー：
1. 会社紹介 + 切り口
2. 課題
3. ソリューション
4. プロダクト / デモ
5. 市場
6. ビジネスモデル
7. トラクション
8. チーム
9. 競合 / 差別化
10. Ask（調達希望）
11. 資金使途 / マイルストーン
12. 付録

Web ネイティブのデックが必要な場合は、このスキルを `frontend-slides` と組み合わせてください。

### ワンページャー / メモ
- 会社の事業内容を簡潔な一文で述べる
- なぜ今なのかを示す
- トラクションと根拠を早い段階で含める
- Ask を明確にする
- 主張を検証しやすくする

### 財務モデル
含めるべき項目：
- 明示的な前提条件
- 必要に応じてベア / ベース / ブルケース
- レイヤーごとの明確な収益ロジック
- マイルストーンに連動した支出計画
- 前提条件が結果を左右する場合の感度分析

### アクセラレーター応募
- 質問されたことに正確に回答する
- トラクション、インサイト、チームの優位性を優先する
- 誇大表現を避ける
- 内部指標をデックおよびモデルと一致させる

## 避けるべきレッドフラグ

- 検証不可能な主張
- 前提条件のない曖昧な市場規模推定
- チームの役割や肩書きの不整合
- 正しく合計されない収益計算
- 前提条件が脆弱な箇所での過度な確信

## 品質チェック

納品前に確認：
- すべての数値が現在の信頼できる情報源と一致している
- 資金使途と収益レイヤーが正しく合計されている
- 前提条件が明示されており、埋もれていない
- 誇大な表現なしにストーリーが明確である
- 最終資料がパートナーミーティングで説明に耐えうる内容である
