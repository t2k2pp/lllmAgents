# コンテンツエンジン

1つのアイデアを、どこにでも同じものを投稿するのではなく、プラットフォームにネイティブな強力なコンテンツに変換する。

## 発動条件

- Xの投稿やスレッドを書く時
- LinkedInの投稿やローンチアップデートを下書きする時
- 短尺動画やYouTube解説のスクリプトを書く時
- 記事、ポッドキャスト、デモ、ドキュメントをソーシャルコンテンツに再利用する時
- ローンチ、マイルストーン、テーマに基づく簡易コンテンツプランを構築する時

## 最初の質問

以下を確認する:
- ソースアセット: 何を元に作成するか
- オーディエンス: ビルダー、投資家、顧客、運営者、または一般層
- プラットフォーム: X、LinkedIn、TikTok、YouTube、ニュースレター、またはマルチプラットフォーム
- ゴール: 認知、コンバージョン、採用、権威構築、ローンチ支援、またはエンゲージメント

## 基本ルール

1. プラットフォームに合わせて適応する。同じコピーをクロスポストしない。
2. 要約よりもフックが重要。
3. すべての投稿に1つの明確なアイデアを持たせる。
4. スローガンよりも具体的な内容を使う。
5. 行動要請は小さく明確にする。

## プラットフォーム別ガイダンス

### X
- 素早く始める
- 投稿あたり、またはスレッドの各ツイートに1つのアイデア
- 必要でない限りリンクを本文に入れない
- ハッシュタグの乱用を避ける

### LinkedIn
- 強い1行目
- 短い段落
- レッスン、結果、要点を明確にフレーミングする

### TikTok / 短尺動画
- 最初の3秒で注意を引く
- ナレーションだけでなく、ビジュアルに基づいたスクリプト
- 1つのデモ、1つの主張、1つのCTA

### YouTube
- 結果を早く見せる
- チャプターで構成する
- 20〜30秒ごとにビジュアルをリフレッシュする

### ニュースレター
- 無関係なアイテムの寄せ集めではなく、1つの明確な視点を提供する
- セクションタイトルはスキャンしやすくする
- オープニング段落に実質的な内容を持たせる

## 再利用フロー

デフォルトのカスケード:
1. アンカーアセット: 記事、動画、デモ、メモ、またはローンチドキュメント
2. 3〜7個のアトミックなアイデアを抽出
3. プラットフォームネイティブなバリエーションを執筆
4. 出力間の重複を削除
5. プラットフォームの意図に合わせてCTAを調整

## 成果物

キャンペーンを求められた場合、以下を返す:
- コアアングル
- プラットフォーム別ドラフト
- オプションの投稿順序
- オプションのCTAバリエーション
- 公開前に必要な不足情報

## 品質ゲート

納品前に:
- 各ドラフトがそのプラットフォームにネイティブに読めること
- フックが強力で具体的であること
- 汎用的な誇大表現がないこと
- 要求がない限りプラットフォーム間でコピーが重複していないこと
- CTAがコンテンツとオーディエンスに合致していること
