# マーケットリサーチ

見せかけのリサーチではなく、意思決定を支えるリサーチを行います。

## 有効化するタイミング

- 市場、カテゴリ、企業、投資家、テクノロジートレンドの調査
- TAM/SAM/SOM の推定値の作成
- 競合や類似プロダクトの比較
- アウトリーチ前の投資家ダシエの準備
- 構築、投資、市場参入前のテーゼの検証

## リサーチ基準

1. 重要な主張にはすべてソースが必要。
2. 最新のデータを優先し、古いデータは明示する。
3. 反論的なエビデンスとダウンサイドケースを含める。
4. 調査結果を単なる要約ではなく、意思決定に変換する。
5. 事実、推論、推奨を明確に分離する。

## 一般的なリサーチモード

### 投資家 / ファンドのデューデリジェンス
収集する情報：
- ファンドサイズ、ステージ、一般的なチェックサイズ
- 関連するポートフォリオ企業
- 公開されたテーゼと最近の活動
- そのファンドが適合する/しない理由
- 明らかなレッドフラグやミスマッチ

### 競合分析
収集する情報：
- マーケティングコピーではなく、プロダクトの実態
- 公開されている場合の資金調達と投資家の履歴
- 公開されている場合のトラクション指標
- 流通と価格設定の手がかり
- 強み、弱み、ポジショニングのギャップ

### 市場規模推定
使用する手法：
- レポートや公開データセットからのトップダウン推定
- 現実的な顧客獲得の前提条件に基づくボトムアップの妥当性チェック
- ロジックの飛躍ごとに明示的な前提条件

### テクノロジー / ベンダー調査
収集する情報：
- 仕組み
- トレードオフと普及のシグナル
- 統合の複雑さ
- ロックイン、セキュリティ、コンプライアンス、運用リスク

## 出力フォーマット

デフォルト構成：
1. エグゼクティブサマリー
2. 主要な発見事項
3. 示唆
4. リスクと留意点
5. 推奨
6. ソース

## 品質チェック

納品前に確認：
- すべての数値にソースが付いているか、推定値と明示されている
- 古いデータにフラグが立てられている
- 推奨がエビデンスから導かれている
- リスクと反論が含まれている
- 出力が意思決定を容易にしている
