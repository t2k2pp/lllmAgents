# 外部設計書 (External Design)

本ドキュメントでは、LocalLLM Agent の外部仕様（ユーザー向け機能・インターフェース・動作要件）について定義します。

## 1. システム概要

LocalLLM Agent は、ローカルで稼働するLLM（大規模言語モデル）を活用した**CLI型AIエージェント**です。ユーザーのPC上で自律的に動作し、ファイルの読み書き、Web検索、ブラウザ操作、コマンドの実行などを通じてタスクを遂行します。Claude Code にインスパイアされた対話型の REPL インターフェースを提供します。

### 1.1 主な特徴
- **完全なローカル・オンプレミス動作**: Ollama, LM Studio, llama.cpp, vLLM といった主要なローカル基盤モデル・プロバイダーに対応
- **多機能なツール群**: ファイル操作をはじめ、Playwright を用いた Web 自動化操作等をプロンプトから自律判定して実行
- **サンドボックスと権限管理**: ユーザーの意図しない破壊的アクションを防止するための3段階（自動許可・要確認・禁止）セキュリティモデルを実装

## 2. ユーザーインターフェース (UI)

### 2.1 REPL コマンドラインUI
エージェントはターミナル上で動作し、コマンドプロンプト形式でユーザーの自然言語入力を受け付けます。
- **マルチライン入力**: ``` （バッククォート3つ）で囲むことで複数行の入力をサポート
- **プロセス中断**: `Ctrl+C` による実行中のタスクのキャンセル

### 2.2 スラッシュコマンド
自然言語のプロンプトに加え、エージェント自体の制御や情報取得のためのスラッシュコマンドを提供します。

| コマンド | 説明 |
|----------|------|
| `/help` | ヘルプや使用可能なコマンド一覧を表示します |
| `/quit` | エージェントを終了します |
| `/clear` | 現在の会話履歴とコンテキストをクリアします |
| `/context` | 現在のコンテキスト（トークン）使用状況を表示します |
| `/setup` | 設定ウィザードを再実行し、LLMプロバイダーやモデルを変更します |
| `/plan` | タスクを事前に分析・設計する「プランモード」を手動で開始します |
| `/skill` | 追加ロードされているスキル（builtin含む）の一覧を表示します |
| `/agents` | バックグラウンド等で稼働しているサブエージェントの一覧・状態を表示します |
| `/status` | 全体の稼働ステータス（コンテキスト・タスク・エージェント）を一括表示します |

## 3. 提供機能とツール群

エージェントはLLMの推論結果に基づき、以下の機能（ツール）を自律的に呼び出します。

### 3.1 ファイル・システム操作
- ファイルの読み取り (`file_read`) と書き込み (`file_write`, 自動ディレクトリ作成対応)
- 既存ファイルの一部書き換え (`file_edit`, 厳密な置換)
- ファイルパターンの検索 (`glob`) と文字列検索 (`grep`)
- ターミナルコマンドの実行 (`bash`, タイムアウト付き)

### 3.2 ブラウザ・Web自動化
- URLからのテキスト抽出 (`web_fetch`)
- Web検索機能 (`web_search`, DuckDuckGo等を利用)
- Playwright統合によるナビゲーション (`browser_navigate`)、クリック (`browser_click`)、テキスト入力 (`browser_type`)
- スクリーンショット取得 (`browser_screenshot`) とアクセシビリティツリー抽出 (`browser_snapshot`)

### 3.3 タスク管理・サブエージェント
- ToDo リストの管理と書き込み (`todo_write`)
- 特定用途に特化した子エージェントの生成・委譲 (`task` ツール)。探索専用 (`explore`) やコマンド特化 (`bash`) など
- ユーザーへの確認・質問 (`ask_user`)

## 4. セキュリティ・権限モデル

ユーザー環境を保護するため、ツールの実行には3段階の権限レベルが設けられています。

1. **自動許可 (auto)**
   - ファイルの読み取りや検索など、システム状態を変更しない安全な操作。自動的に実行されます。
2. **ユーザー確認 (ask)**
   - ファイルの書き込み/編集、コマンド実行、ブラウザ操作など、システム状態を変更する、または外部リソースにアクセスする操作。
   - 実行前にユーザーへ許可（今回のみ/常に許可/拒否）を求めます。
3. **禁止 (deny)**
   - システムを破壊する可能性がある危険なコマンド（`rm -rf /` やフォーマット等）は自動的に検知し、実行をブロックします。

また、ファイル操作は設定されたドキュメントルートやホームディレクトリ等に限定される**サンドボックス機能**が備わっています。

## 5. 設定と環境

- **設定ファイル**: `~/.localllm/config.json`
- **動作要件**: Node.js 18以上、ローカルLLMサーバー（Ollama等）が稼働している環境、Playwright（ブラウザ機能利用時のみ）
