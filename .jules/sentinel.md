## 2024-03-07 - Command Injection in grep tool
**Vulnerability:** The `grep` tool was using `execSync(args.join(" "))` to execute ripgrep commands. This constructed a single string passed to `execSync`, which implicitly spawns a shell (like `/bin/sh`) to parse and execute the command. This allowed an attacker or LLM to inject arbitrary shell commands via the `pattern`, `path`, or `glob` parameters (e.g., using `;` or `&&`).
**Learning:** Using `execSync` (or `exec`) with a dynamically joined string argument creates a shell context where input parameters can break out of their intended arguments and execute arbitrary shell commands. It is a classic Command Injection vector.
**Prevention:** Always use `execFileSync` or `spawn` without a shell option when executing commands with user-provided arguments. Pass arguments as an array (`execFileSync(cmd, args)`) instead of a concatenated string. This ensures arguments are passed directly to the executable without shell parsing.
