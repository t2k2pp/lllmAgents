---
# 概要
ユーザーの要望（「〇〇をするスキルを作って」など）を元に、ローカルLLM自身が利用可能なツールを活用して自動で新しいスキル（Markdownファイル）を作成し、追加するメタスキルです。
このスキル自体が、Claude公式のスキル作成ベストプラクティスに則ってスキルファイルを出力する役割を持ちます。

# スキルの保存場所とファイルフォーマット
ローカルLLMのエージェントシステム（lllmAgents）では、スキルはローカルディレクトリの `.localllm/skills/` に Markdown 形式で保存されます。
ファイルは必ず以下の YAML フロントマターを含める必要があります。

\`\`\`yaml
---
name: スキルの表示名 (短く分かりやすく)
description: スキルの一言説明
trigger: /任意のコマンド名
---
# ここから下はLLMに向けたプロンプト（指示書）になります。

# 概要
このスキルが何を行うのかを定義します。

# 実行手順
1. 引数をどう受け取るか（例：`/コマンド 引数` または `ask_user` ツールの利用）
2. どのツールを使うか（`file_read`, `browser_navigate`, `bash` など）の具体的な手順
3. エラー時の処理や保存先などの制約事項
\`\`\`

# 実行手順
1. ユーザーからのプロンプトの内容（作りたいスキルの要件）を詳細に解析します。
2. そのスキルを実現するために、ローカルLLMが持つどのツールを使用すべきか（ファイル操作、ブラウザ操作、実行環境など）を決定します。
3. スキルの名前、説明、トリガーコマンド（例: `/screenshot`）を決定します。
4. スキル指示書となるMarkdownコンテンツを構築します。ベストプラクティスとして「簡潔な手順」「条件分岐」「システムプロンプトの見本」を組み込んでください。
5. `file_write` ツールを使用して、プロジェクト直下の `.localllm/skills/{ファイル名}.md` として対象スキルを出力・保存します。ディレクトリが存在しない場合は作成含めて行ってください。
6. 出力が完了したら、ユーザーに対して「作成したスキルのコマンド（例: `/xxx`）を実行すると使えるようになったこと」を案内してください。
