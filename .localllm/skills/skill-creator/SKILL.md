# Skill Creator

## 概要

このスキルは、LLM自身がユーザーの要望に基づいて「堅牢で決定論的なスキル」を自動的に作成・追加するためのメタ・ツールです。
単なるプロンプト集ではなく、「手続き・スクリプト・リファレンスをバンドルしたモジュール」としてのスキルを作成します。

## ベストプラクティス（コア原則）

1. **Low Freedom を目指す**: LLMはツール(ブラウザやbashなど)をその場で試行錯誤するのではなく、決定論的なスクリプト(`scripts/xxx.py` や `scripts/xxx.js`)を作成し、スキルはそのスクリプトを呼び出すだけにするべきです。
2. **Progressive Disclosure**: `SKILL.md` は最小限の実行手順のみを残し、具体的なコンテキストや大量のサンプルコードは `references/` に分離します。
3. **明確な役割分担**:
   - `SKILL.md`: ツールのトリガーと使用手順
   - `scripts/`: スキルの心臓部。完全自動化・エラーハンドリングされた確実なスクリプト

## スキル作成の手順 (Low Freedom Workflow)

ユーザーの要望（「〇〇をするスキルが欲しい」等）を受け取ったら、以下の手順でスキルを作成します。

1. **スキルの構成要素を決める**: `scripts/` に配置する実行スクリプトが必要か、`references/` が必要か検討します。
2. **Bashスクリプトによる初期化**:
   `bash` ツールを使用して、以下のスクリプトを実行し、ディレクトリとテンプレートを生成します。
   
   ```bash
   node .localllm/skills/skill-creator/scripts/init_skill.js <スキル名(ケバブケース)>
   ```

3. **出力されたファイル群の編集**:
   - 生成された `.localllm/skills/<スキル名>/SKILL.md` を `file_edit` や `file_write` を用いて、ユーザー要件に沿って編集します。トリガーとなるコマンド（例: `/my-skill`）を設定します。
   - `.localllm/skills/<スキル名>/scripts/` 内に、専用のスクリプト（JSやPython, Bash等）を `file_write` によって作成します。
4. **動作テスト**:
   - ユーザーからの入力例などを想定して、作成した `scripts/` 内のスクリプトを `bash` ツールで試運転します。
5. **ユーザーへの結果報告**:
   - スキル作成が完了したこと、トリガーコマンド（/コマンド）と使い方の例を説明して終了します。
